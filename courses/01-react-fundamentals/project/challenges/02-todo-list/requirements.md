# Challenge 02: Technical Requirements

## Functional Requirements

1. ✅ Component must be named `TodoList`
2. ✅ Must manage array of todos with state
3. ✅ Each todo must have: `id`, `text`, `completed` properties
4. ✅ Must be able to add new todos
5. ✅ Must be able to toggle completion status
6. ✅ Must be able to delete todos
7. ✅ Completed todos must be visually distinct
8. ✅ Must use proper React state management

## Code Quality Requirements

1. ✅ Must use TypeScript with proper types
2. ✅ Todo interface/type must be defined
3. ✅ Must handle edge cases (empty input, etc.)
4. ✅ Code must pass ESLint
5. ✅ No console.log statements

## Architecture Requirements

1. ✅ Component in `src/components/TodoList.tsx`
2. ✅ Must use useState for state management
3. ✅ Must use proper array methods (map, filter)
4. ✅ Event handlers must be properly defined
5. ✅ Must use controlled components for input

## Scoring

- Functional correctness: 40%
- Code quality: 25%
- Architecture: 20%
- Best practices: 10%
- AI review: 5%
